How it works:

  • show #content - Makes the element visible by removing display: none
  • hide #content - Hides the element by adding display: none
  • toggle .hidden on #content - Toggles the .hidden class (which sets display: none)

Key Concepts:

  • Simple Commands: show, hide, and toggle are built-in commands
  • CSS Selectors: Target any element using standard CSS selectors
  • Declarative: No need to manage display state manually
  • Animation-Ready: Combine with transitions for smooth effects

Common Use Cases:

  • Modals: Show/hide dialog boxes
  • Dropdowns: Toggle menu visibility
  • Accordions: Expand/collapse sections
  • Notifications: Display and dismiss alerts
  • Loading States: Show/hide loading indicators

Toggle Command Syntax:

The toggle command supports CSS classes and attributes:

<!-- Toggle CSS classes (recommended for visibility) --> <button _="on click toggle .active on #element">Toggle Class</button> <button _="on click toggle .hidden on #element">Toggle Hidden</button> <button _="on click toggle .visible on #element">Toggle Visible</button> <!-- Toggle attributes --> <button _="on click toggle @disabled on #submit-btn">Enable/Disable</button> <button _="on click toggle [@disabled=\"true\"] on #submit-btn">Toggle Value</button>

Advanced Patterns:

You can also show/hide multiple elements at once:

<!-- Show all items with class 'item' --> <button _="on click show .item">Show All</button> <!-- Hide all except the first --> <button _="on click hide .item then show .item:first-child"> Show Only First </button> <!-- Toggle for a duration --> <button _="on click toggle .active on #element for 2s"> Flash Active </button>

Try it yourself:

  • Add more elements and control them independently
  • Use class selectors (.className) to show/hide groups
  • Combine with wait 2s for timed visibility changes
  • Chain commands: hide #loading then show #content
