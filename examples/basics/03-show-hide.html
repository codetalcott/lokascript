Show/Hide Elements

Control element visibility
Two approaches: built-in commands or class-based toggling.
show hide toggle

Demo 1: Show/Hide Commands

The show and hide commands manipulate the display style directly. Use these when you only need show or hide (not toggle).

For toggling, use a CSS class. All three buttons work on the same .hidden class, so they stay in sync.

The Code

Approach 1: Show/Hide Commands

<!-- Direct visibility commands --> <button _="on click show #content">Show</button> <button _="on click hide #content">Hide</button>

Approach 2: Class-based (with Toggle)

<!-- Class manipulation for consistent toggling --> <button _="on click remove .hidden from #content">Show</button> <button _="on click add .hidden to #content">Hide</button> <button _="on click toggle .hidden on #content">Toggle</button> <!-- CSS --> .hidden { display: none; }

When to use each approach:

  • Show/Hide commands: Simple cases where you only need show or hide
  • Class-based: When you need toggle, or want CSS transitions

Why class-based for toggle?

The show/hide commands and toggle .hidden use different mechanisms (inline styles vs CSS class). Mixing them causes inconsistent state. Use one approach consistently.

Common patterns:

<!-- Chain commands --> <button _="on click hide #loading then show #content">Load</button> <!-- Toggle with duration --> <button _="on click toggle .highlight on #item for 2s">Flash</button> <!-- Target multiple elements --> <button _="on click show .panel">Show All Panels</button>
