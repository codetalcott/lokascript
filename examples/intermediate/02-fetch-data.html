The Code

Basic Fetch:

<button _="on click fetch https://api.example.com/data as json put result into #output"> Load Data </button>

With Loading State:

<button _="on click add .loading to me show #spinner fetch https://api.example.com/users as json put result into userData -- Process the data hide #spinner remove .loading from me"> Fetch </button>

Loop Through Results:

<button _="on click fetch https://api.example.com/posts as json put result into posts set html to '' repeat for post in posts set html to html + `<div>${post.title}</div>` end put html into #list"> Load Posts </button>

Error Handling:

<button _="on click try fetch https://api.example.com/data as json put result into #output catch error put 'Error: ' + error into #error-message show #error-box end"> Load </button>

How it works:

  • fetch URL as json - Makes HTTP GET request and parses JSON response
  • put result into variable - Stores the response data
  • repeat for item in items - Loops through array results
  • try/catch - Handles errors gracefully
  • Template literals (`${variable}`) - Embed data in HTML

Key Concepts:

  • Async Operations: Fetch automatically waits for response
  • JSON Parsing: as json converts response to JavaScript object
  • Result Variable: result contains the fetched data
  • Template Literals: Use backticks for string interpolation
  • Error Handling: try/catch for network errors

Fetch Options:

  • as json - Parse JSON response
  • as text - Get plain text response
  • as html - Get HTML response
  • as response - Get raw Response object

HTTP Methods:

<!-- POST request --> fetch /api/users method: 'POST' headers: { 'Content-Type': 'application/json' } body: JSON.stringify({ name: 'John', email: 'john@example.com' }) as json <!-- PUT request --> fetch /api/users/1 method: 'PUT' body: JSON.stringify({ name: 'Updated Name' }) as json <!-- DELETE request --> fetch /api/users/1 method: 'DELETE'

Common Use Cases:

  • Load User Data: Fetch and display profile information
  • Blog Posts: Load and render article lists
  • Search Results: Fetch filtered data based on input
  • Form Submission: POST data to server
  • Auto-complete: Fetch suggestions as user types
  • Pagination: Load more items on button click
